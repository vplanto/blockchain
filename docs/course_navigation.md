# Навчальний посібник: Структура курсу та навігація

Для успішного опанування матеріалу курсу зверніть увагу на структуру навчальних лекцій та правила роботи з практичними ресурсами.

### 1. Структура кожної лекції

Кожна лекція у цьому курсі побудована за єдиним методичним шаблоном, орієнтованим на практичне проектування:
1. **Розминка (експрес-опитування):** Короткі концептуальні питання на початку лекції для самоперевірки та актуалізації опорних знань.
2. **Теоретичний блок:** Фундаментальні основи, математичні моделі та алгоритми розподілених систем.
3. **Інженерні кейси та демонстрації (Must Do / Практика):** Розбір реальних архітектурних рішень, життєвих циклів транзакцій, вразливостей (наприклад, The DAO, Parity Wallet) та взаємодії з симуляторами консенсусу.
4. **Рекомендована література:** Добірка офіційних інженерних стандартів (ERC-специфікації), Whitepapers (Satoshi Nakamoto, Vitalik Buterin) та наукових статей для глибокого вивчення теми (зібрана в кінці всього курсу).
5. **Питання та відповіді (екзаменаційний блок):** Перелік ключових залікових/екзаменаційних питань з короткими відповідями для підготовки до контрольних заходів.
6. **Інженерний міні-кейс (на 1 бал):** Практична задача в кінці лекції для самостійного аналізу та закріплення матеріалу.

### 🔗 Робота з інтерактивними матеріалами та кодом на GitHub

Протягом курсу ви зустрінете посилання на вихідний код у форматі `(код на GitHub)`. Усі ці ресурси знаходяться у публічному репозиторії курсу: **[github.com/vplanto/blockchain](https://github.com/vplanto/blockchain)**. 

Щоб полегшити навігацію, ознайомтеся зі структурою папок репозиторію:
* **`/solidity/`** (посилання на приклади кодів) — містить готові до використання смарт-контракти для середовища Remix IDE та локального фреймворку Foundry.
* **`/docs/workshops/`** — покрокові інструкції до практичних робіт та індивідуальних завдань.
* **`/docs/consensus/`** — файли симуляторів консенсусу (pBFT, Paxos, Raft, Byzantine Generals тощо), які можна відкривати безпосередньо у браузері або розгортати локально.
* **`/docs/images/`** — архітектурні схеми та ілюстрації, використані у лекціях.

Якщо ви працюєте з офлайн-версією навчального посібника (PDF), ви можете використовувати інтерактивні посилання на GitHub Pages (інтерактивні рендери), які працюють без необхідності встановлення локального оточення: **[vplanto.github.io/blockchain/](https://vplanto.github.io/blockchain/)**.
