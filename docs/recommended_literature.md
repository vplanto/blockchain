# Рекомендована література

Не читайте перекази блогерів. Читайте інженерну документацію та першоджерела.

### Фундаментальні основи та розподілені системи

1.  **[Bitcoin Whitepaper (Satoshi Nakamoto)](https://bitcoin.org/bitcoin.pdf)** — оригінальний опис першої децентралізованої платіжної системи Bitcoin від Сатоші Накамото. Обов'язково до прочитання.
2.  **[Brewer's Conjecture Proof (Gilbert & Lynch, 2002)](https://users.ece.cmu.edu/~adrian/731-sp04/readings/GL-cap.pdf)** — математичне доведення CAP-теореми (Feasibility of Consistent, Available, Partition-Tolerant Web Services).
3.  **[Ethereum Yellow Paper (Gavin Wood)](https://ethereum.github.io/yellowpaper/paper.pdf)** — математично строгий опис роботи EVM та структури стану Ethereum.

### Розробка смарт-контрактів та екосистема

4. **[Foundry Book](https://book.getfoundry.sh)** — офіційна інженерна книга та документація фреймворку Foundry для локального тестування смарт-контрактів.
5. **[ERC-20 Token Standard Spec](https://eips.ethereum.org/EIPS/eip-20)** — офіційний стандарт та специфікація ERC-20 інтерфейсів взаємозамінних токенів.
6. **[ERC-721 Non-Fungible Token Standard Spec](https://eips.ethereum.org/EIPS/eip-721)** — офіційний стандарт та специфікація ERC-721 інтерфейсів невзаємозамінних токенів (NFT).
7. **[EIP-1167: Minimal Proxy Contract](https://eips.ethereum.org/EIPS/eip-1167)** — специфікація та опис роботи мінімальних Proxy-контрактів (шаблон оновлюваних смарт-контрактів).
